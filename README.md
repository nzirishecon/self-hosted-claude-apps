# Self-Hosted Claude Apps

This repository provides a simple way to host web applications generated by Anthropic's Claude AI on GitHub Pages. It includes a landing page that displays all your apps as clickable tiles, making it easy to organise and access multiple applications in one place.

## Overview

When Claude is used to generate web applications, the output is typically in JSX (JavaScript XML) format, which must be transpiled into standard HTML, CSS and JavaScript before it can execute in a web browser. This repository solves that problem by:

- Providing a template structure for each app.
- Using GitHub Actions to automatically build and deploy your apps.
- Creating a central landing page where all your apps are listed.

The repository is designed for users who use Claude to generate application code, and need a tool to handle the build and hosting process. The code and GitHub Actions workflow in this repository were generated by Claude before being reviewed. This was necessary in particular with the GitHub Actions workflow as Claude did not initially specify the latest versions for each step in the workflow and there were bugs with the step named `Place app builds into _site/apps/`. Other edits were made to the workflow and these have been labelled with comments.

## How the Repository Works

### Repository Structure

- `index.html`: The main landing page that shows tiles for all your apps.
- `apps/`: Directory containing each individual app in its own subdirectory.
- `examples/`: Contains examples of apps generated by Claude. Each app directory includes a `README.md` file containing the prompt that was issued to Claude to generate the app.
- `.github/workflows/deploy.yml`: GitHub Actions workflow that builds and deploys everything.

### App Structure

Each app lives in its own directory under `apps/` and contains these files:

- `src/App.jsx`: The main app component (generated by Claude).
- `src/main.jsx`: Entry point that renders the app.
- `app.config.json`: Configuration file with app details.
- `index.html`: HTML template for the app.
- `package.json`: Dependencies and build scripts.
- `vite.config.js`: Build configuration.

### Build and Deployment Process

When you push changes to the main branch:

1. GitHub Actions scans the `apps/` directory for apps (any directory with a `package.json`).
1. For each app, it runs `npm install` to install dependencies, then `npm run build` to create the final HTML/CSS/JS files.
1. All built apps are collected into a `_site` directory.
1. A `registry.json` file is generated from each app's `app.config.json`.
1. Everything is deployed to GitHub Pages.

The landing page reads `registry.json` to display tiles linking to each app.

## Deploying This Repository to GitHub

### Step 1: Get the Repository

There are two methods:

**Option A: Fork the repository**
1. Go to https://github.com/bakkerjarr/self-hosted-claude-apps.
1. Click the "Fork" button in the top right.
1. This creates a copy in your GitHub account.

**Option B: Create from scratch**
1. Create a new repository on GitHub.
1. Clone this repository to your computer: `git clone https://github.com/bakkerjarr/self-hosted-claude-apps.git` OR `git clone git@github.com:bakkerjarr/self-hosted-claude-apps.git`
1. Change the remote to point to your new repository: `git remote set-url origin https://github.com/your-username/your-repo.git`
1. Push to your repository: `git push -u origin main`

### Step 2: Configure GitHub Actions

1. In your GitHub repository, go to `Settings` → `Actions` → `General`.
1. Under `Actions permissions`, select the radio button for **Allow bakkerjarr, and select non-bakkerjarr, actions and reusable workflows** then tick the box for **Allow actions created by GitHub**. Click on the `Save` button in this section.
1. Under `Approval for running fork pull request workflows from contributors`, select the radio button for **Require approval for all external contributors**. Click on the `Save` button in this section.

### Step 3: Enable GitHub Pages

1. In your GitHub repository, go to `Settings` → `Pages`.

> Be sure to check the visibility settings for your repository under `Settings` → `General`. If you have a free/unlicensed GitHub account then GitHub Pages can be enabled for **public** repositories. 

2. Under `Source`, select `GitHub Actions`.
3. The repository is already configured with the necessary workflow, so no additional setup is needed.

### Step 4: Visit you unpublished site

Your site will be available at: `https://your-username.github.io/repository-name/`

## Deploying Additional Apps

### Step 1: Generate an App with Claude

Use this prompt with Claude:

>[Provide a description of what you want the app to do.]
>
>The app will be deployed using an existing GitHub Actions pipeline. I expect there to be the following files:
>* src/App.jsx
>* src/main.jsx
>* app.config.json
>* index.html
>* package.json
>* vite.config.js
>
>The app will be accessed from a landing page where a collection of tiles summarise multiple apps. Each app is contained in its own directory in a single repository.
>
>The schema for the `app.config.json` file is:
>
>```
>{
>  "name": "Name of the app.",
>  "description": "A short description of the app, adding a note that commentary has been made by AI.",
>  "icon": "An icon for the app.",
>  "category": "A category tag for the app.",
>  "updated": "When the statistics behind the app were last updated."
>}
>```

The required files are generated automatically.

### Step 2: Create App Directory

1. In your repository, create a new directory under `apps/` (e.g., `apps/my-new-app/`)
1. Place all the files Claude generated into this directory using the following structure:

- `apps/my-new-app/src/App.jsx`: The main app component (generated by Claude).
- `apps/my-new-app/src/main.jsx`: Entry point that renders the app.
- `apps/my-new-app/app.config.json`: Configuration file with app details.
- `apps/my-new-app/index.html`: HTML template for the app.
- `apps/my-new-app/package.json`: Dependencies and build scripts.
- `apps/my-new-app/vite.config.js`: Build configuration.

### Step 3: Configure the App

Ensure that the `app.config.json` files matches the schema:

```json
{
  "name": "Name of the app",
  "description": "A short description of the app, adding a note that commentary has been made by AI.",
  "icon": "An icon for the app.",
  "category": "A category tag for the app",
  "updated": "When the statistics behind the app were last updated."
}
```

Example:
```json
{
  "name": "NZ Population Change",
  "description": "An interactive breakdown of New Zealand's population growth and ethnic composition across the 2006, 2013, 2018, and 2023 censuses. AI-generated commentary is included for context.",
  "icon": "📊",
  "category": "Demographics",
  "updated": "2024"
}

```

### Step 4: Deploy

1. Commit and push your changes: `git commit -am "COMMIT MESSAGE" && git push`
1. GitHub Actions will automatically build and deploy your app.
1. Visit your GitHub Pages site to see the new app tile on the landing page.

## Removing Apps

To remove an app:
1. Delete the app's directory from `apps/`.
1. Commit and push the changes.
1. The app will no longer appear on the landing page.

## Troubleshooting

### The GitHub Actions workflow completed with no errors but one or more apps don't open when you try to access them from the landing page.

The Actions workflow completing without error only indicates that the workflow was able to compile the apps and deploy them to GitHub Pages. If an app doesn't open then this can indicate that there is an issue with the app code itself. Check the following:

1. Ensure that Claude outputted the required files and that they match directory structure below (relative to the root directory for the repository):
- `apps/[APP NAME]/src/App.jsx`: The main app component (generated by Claude).
- `apps/[APP NAME]/src/main.jsx`: Entry point that renders the app.
- `apps/[APP NAME]/app.config.json`: Configuration file with app details.
- `apps/[APP NAME]/index.html`: HTML template for the app.
- `apps/[APP NAME]/package.json`: Dependencies and build scripts.
- `apps/[APP NAME]/vite.config.js`: Build configuration.

2. Ensure that the `vite.config.js` file for the app only contains the following code:

```javascript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  base: './',
})
```
